# 프로젝트 요약: 글로벌 증시 거래시간 캘린더 자동화

## 1. Project Goal
- Next.js, FullCalendar, Tailwind CSS를 활용한 글로벌 증시 캘린더 웹 애플리케이션 개발
- 선택한 타임존(Timezone)에 맞추어 5개국(미국, 한국, 일본, 중국, 영국)의 증시 세션을 자동 계산 및 시각화
- 서머타임(DST) 적용 국가의 경우 날짜에 따른 세션 시간 동적 변경 지원
- API 기반 실시간 휴장일 데이터 연동 및 캘린더 내 시각적 예외 처리(휴장 뱃지, 팝오버)

## 2. Core Rules (엄수 지침)
- **NXT 마켓 및 데이마켓 적용 범위**: 한국 브로커(증권사)를 통한 미국 주식 거래 시간에 한정하여 적용
- **타임존 렌더링**: `date-fns-tz`의 `formatInTimeZone`을 사용하여 브라우저 로컬 시간이 아닌 유저가 선택한 타임존에 맞춰 세션 시간을 렌더링
- **서머타임(DST) 동적 계산**: 
  - 미국: 3월 2번째 일요일 ~ 11월 1번째 일요일
  - 영국: 3월 마지막 일요일 ~ 10월 마지막 일요일
- **UI/UX 렌더링 제약**: 
  - 달력 리렌더링 시 React 렌더링 사이클(`flushSync` 에러) 방지를 위해 `changeView`, `gotoDate` 등은 `setTimeout(..., 0)`으로 비동기 호출
  - 휴장/개장 상세 정보는 뷰를 가리는 거대한 모달 대신, 마우스 클릭 좌표(`clientX`, `clientY`) 기반의 소형 팝오버(Popover)로 구현
  - 일간(Day) 뷰에서는 슬롯 높이를 `0.8em`으로 압축하여 스크롤을 방지하고, 툴팁 대신 텍스트 블록 안에 시간을 직접 명시
- **보안 원칙**: Github 업로드 시 `.env*` 등 민감 파일이 유출되지 않도록 `.gitignore` 적용 필수 확인

## 4. 버그 및 이슈 리포트 & History
- **[Fix] FullCalendar 타임존 툴팁 동기화 오류**: 기본 브라우저 로컬 타임으로 툴팁 시간이 뜨는 문제 발생 -> `date-fns-tz` 도입 및 이벤트 렌더링 시 선택된 타임존으로 포맷팅하도록 수정
- **[Fix] React flushSync 충돌**: 달력 뷰(월/주/일) 전환이나 날짜 점프 시 라이프사이클 충돌 에러 발생 -> `calendarApi` 호출부를 `setTimeout` 기반 비동기 처리로 감싸 해결
- **[UI/UX] 툴팁 반응성 지연 및 시야 가림 현상 개선**: 브라우저 기본 `title` 속성 대신 `getBoundingClientRect()`을 활용하여 마우스 커서에 즉각 반응하는 커스텀 HTML 툴팁 구현
- **[UI/UX] 데이 뷰(Day View) 레이아웃 최적화**: 24시간 표기로 인한 스크롤 낭비 방지를 위해 `globals.css`에서 해당 뷰 슬롯 높이를 `0.8em`으로 대폭 축소 및 블록 내 시간 직접 표기
- **[UI/UX] 모달 창 공간 낭비**: 휴장일 뱃지 클릭 시 화면 전체를 가리는 모달 창 대신, `clientX/clientY` 기반의 마우스 오버레이 팝오버 렌더링 방식으로 전면 개편
- **[Fix] W버튼(주차 이동) 레이아웃 쏠림 현상**: 날짜 옆에 버튼을 넣으면서 정렬이 무너짐 -> 날짜는 정중앙(`relative`)에 고정하고 버튼은 `absolute` 트릭을 사용하여 문서 흐름에서 제외시켜 완벽한 중앙 정렬 달성
- **[Feature] 모바일 전용 UI(MobileMarketView) 도입**: 모바일 디바이스 감지 시 별도의 반응형 레이아웃 렌더링. 데이 뷰에서 카드형식과 시간대별 타임라인 보기 스위칭 지원.
- **[UI/UX] W버튼 및 뱃지 중앙 정렬 완벽 보정**: 모바일 화면에서 '오늘/이번주/이번달 뱃지'와 '휴장 안내 버튼'을 `absolute`를 활용해 문서 흐름에서 제외시켜 텍스트 중앙 정렬 유지.
